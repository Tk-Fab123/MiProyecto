CAMPO 4 - Mejores prácticas y comandos avanzados

1. Convención de nombres
Las ramas deben tener nombres claros:
- feature-login
- cambios-fabian
- mejoras-amigo

2. Mensajes de commit
Deben ser claros y descriptivos:
- Agregar campo 4
- Corregir errores
- Actualizar documentación

3. Importancia de branches
Permiten trabajar sin afectar la rama principal (main).

4. Estrategia Git Flow
- main: versión estable
- develop: desarrollo
- feature: nuevas funciones
- hotfix: correcciones rápidas

5. git stash
Guarda cambios temporalmente sin hacer commit.

6. git revert
Revierte cambios sin borrar el historial.

7. git cherry-pick
Copia commits específicos de otras ramas.
